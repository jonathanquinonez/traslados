Tables

Tables are a nice way to organize a lot of data. We provide a few utility classes to help you style your table as easily as possible. In addition, to improve mobile experience, all tables on mobile-screen widths are centered automatically.

Need just simple table without any custom feature. Go ahead and choose this style.

Alan Jellybean $3.76
Jonathan Lollipop $7.00

Datatable with Paginate

Want to have pagination. Set the default class .mat-datatable and play with Datatable Script. Refer the markup below this preview.


$('table.datatable-pagination').DataTable({
  pagingType: "simple",
  language: {
    paginate: {'next': 'Next &rarr;', 'previous': '&larr; Prev'}
  }
});
Demo

Datatable Save state

DataTables has the option of being able to save the state of a table (its paging position, ordering state etc) so that is can be restored when the user reloads a page, or comes back to the page after visiting a sub-page. This state saving ability is enabled by the stateSave option.


$('.datatable-save-state').DataTable({
  stateSave: true
});
Demo

Datatable - Scrollable

DataTables has the ability to show tables with horizontal scrolling, which is very useful for when you have a wide table, but want to constrain it to a limited horizontal display area.


$('.datatable-scroll-y').DataTable({
  autoWidth: true,
  scrollY: 300
});

Datatable Ordering

With DataTables you can alter the ordering characteristics of the table at initialisation time. Using the order initialisation parameter, you can set the table to display the data in exactly the order that you want. You can set default order field by defining your defaulf sort column.


$('.datatable-sorting').DataTable({
  order: [3, "desc"]
});

Multi column ordering can be achieve as follow.


$('.datatable-multi-sorting').DataTable({
  columnDefs: [{
    targets: [0],
    orderData: [0, 1]
  }, {
    targets: [1],
    orderData: [1, 0]
  }, {
    targets: [4],
    orderData: [4, 0]
  }, {
    orderable: false,
    width: '100px',
    targets: [5]
  }]
});

Order direction sequence control can be achieve as follow.

Column 1 - default ordering
Column 2 - default ordering
Column 3 - ascending ordering only
Column 4 - descending ordering, followed by ascending and then ascending again
Column 5 - descending ordering only
Column 6 - default ordering

$('.datatable-sequence-control').dataTable( {
  "aoColumns": [
    null,
    null,
    {"orderSequence": ["asc"]},
    {"orderSequence": ["desc", "asc", "asc"]},
    {"orderSequence": ["desc"]},
    null
  ]
});

Datatable Page length options

The configuration options offered by DataTables extend much further than the options shown in the basic initialisation of this documentation. Through combinations of the options available and the use of callbacks, DataTables is completely customisable and will fit into exactly what you need for your table display.


$('.datatable-show-all').DataTable({
  lengthMenu: [[10, 25, 50, -1], [10, 25, 50, "All"]]
});

DOM positioning

When customising DataTables for your own usage, you might find that the default position of the feature elements (filter input etc) is not quite to your liking.

l - Length changing
f - Filtering input
t - The Table!
i - Information
p - Pagination
r - pRocessing
< and > - div elements
<"#id" and > - div with an id
<"class" and > - div with a class
<"#id.class" and > - div with an id and class

$('.datatable-dom-position').DataTable({
  dom: '<"datatable-header length-left"lp><"datatable-scroll"t><"datatable-footer info-right"fi>',
});
