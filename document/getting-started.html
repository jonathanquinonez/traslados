Get Started

Download

Materialized Forge Admin Template downloable includes soruce files, gulpfile.js, package.json. You will have 4 kind of menus in this theme. Two different vertical menus and two different horizontal menus. Apart from this a lot extensible components and plugins will enhance your development. Forge comes in two different forms. You can select which version you want depending on your preference and expertise. To start using Materialize, all you have to do is download one of the options below.

Standard

This is the standard version that comes with both the minified and unminified CSS and JavaScript files. This option requires little to no setup. Use this if you are unfamiliar with Sass.

Sass

This version contains the source SCSS files. By choosing this version you have more control over which components to include. You will need a Sass compiler if you choose this option.

Sidebar Light/Dark

We have two kind of sidebar. One is light and another is dark. User can change the scheme by importing sb-light for light sidebar and sb-dark for dark sidebar in _varible.scss. By default it will be sb-dark.

NPM Commands

Here, we assume that Node has been installed on your machine. You have also setup Gulp-cli along with Node.

We have used GULP for module bundling. For fast standard setup, you can execute below command in your terminal.


npm start

Apart from this, you can refer below table of commands for specific task.

Command Usage
npm start This includes concurrent execution of task. It includes assets generation and prepare dist folder and prepare for production mode. This bash command can be used for very initial stage.
npm run gulp:assets It will concentrate only on assets bundling which majorly copying thing from source(src) to destination(dist).
npm run gulp:task

It can be used for gulp default task execution.It includes important and repeated useful context of application development.

This compile copy:fonts, scss, scripts, templates, css-plugins.
npm run gulp:w

Watch out scss and pug files for any updates.
npm run gulp:dev

This is combination of default task and also watch. Once both are ready then it will preprare a server for you.
npm run gulp:prod

It will prepare the default tasks and make ready all components and minify the css files.
gulp scss Prepare main.css file from ./src/scss/** files.
gulp mis:scss Prepare dynamic.min.css for dyamic classes used in the theme. markup.css for demondtration of code for element/component.
gulp css-plugins Process only scss plugins for any updates.
gulp minify-css Just minify main.css file at same location.
gulp scripts Process scripts after completion of script:es6 and provide all.js as product.
gulp script:es6 Perform babel opeation on JS and provide browser supported JS from EcmaScript.
gulp templates Provide *.html file as output from *.pug for pug/pages folder.
gulp templates --type dev Reflect internal elements update. Ex: Layouts, nav, included file.
gulp download:JS Download extenal JS and store for distribution.
gulp copy:plugins Make a copy of plugins for distribution.
gulp copy:chartJS Import ChatJS from node_modules and copy to distribution.
gulp copy:fonts Make a copy of fonts to distribution.
gulp copy:js Make a copy of JS to distribution.
gulp copy:images Make a copy of Images to distribution.
gulp serve To make browserSync server available.
gulp watch Watch on gulp task defined in gulpfile.js to any updates.
gulp Execute default configuration for distribution. Identical to npm run gulp:task
gulp assets It will concentrate only on assets bundling which majorly copying thing from source(src) to destination(dist). Just identical to npm run gulp:assets
